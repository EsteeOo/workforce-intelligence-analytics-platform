# Data Model Plan

## Objective
Define the structure, grain, and relationships of all datasets before data generation and transformation.

## Core Principles
- All fact tables will have a clearly defined grain
- Dimension tables will standardise shared attributes
- Relationships will support analytical slicing across workforce domains
- Data design will intentionally introduce variation, imbalance, and realistic patterns

## Planned Domains
- Workforce (HRIS)
- Hiring (ATS)
- Learning (LMS)
- Engagement
- Workforce Operations (Timekeeping)

- table grain
- key fields
- relationships
- data behaviour patterns
